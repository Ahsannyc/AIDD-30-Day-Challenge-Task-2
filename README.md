Q2. Vide Coding vs Spec-driven development 

Vibe-coding is like building a project what ever we feel right at the moment, like our gut feelings but it becomes difficult to grow the project since we cant properly add up new ideas into it properly, since no planning or thinking was involved, so no project structure, no architecture, it is hard to extend or change 

 

But in spec-driven development, we have a clear direction, like a blue print,  with detailed instructions before we even start a project. So we have a clear requirements and a plan before coding begins. We write down every detail we want. So, we can easily add new features or fix things later.
